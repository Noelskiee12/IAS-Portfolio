AIS NOTES

NETWORK TRAFFIC ANALYSIS

— The process of capturing, inspecting, and analyzing data flow across a network. Helps identify common protocols, baseline activity, anomalies, and threats.

OSI MODEL (QUICK PRIMER)

L7 – Application: HTTP, FTP

L6 – Presentation: SSL, TLS

L5 – Session: Connection management

L4 – Transport: TCP, UDP

L3 – Network: IP, routing

L2 – Data Link: Ethernet frames

PURPOSES OF TRAFFIC ANALYSIS

— Understand and characterize ports & protocols used.

— Establish a normal baseline for the network.

— Detect, monitor, and respond to threats or anomalies.

— Gain deep insight into network behavior.

COMMON TOOLS

— tcpdump – CLI packet capture

— Tshark – CLI version of Wireshark

— Wireshark – GUI packet analyzer (most used)

— NGrep – Pattern matching for packets

— tcpick – TCP stream monitoring

— Network Taps & SPAN Ports – Copy traffic for analysis

TRAFFIC ANALYSIS PROCESS

— Ingest Traffic – Capture data (with optional filters).

— Reduce Noise – Filter out unnecessary packets.

— Analyze & Explore – Focus on relevant packets.

— Detect & Alert – Identify malicious vs. normal traffic.

TYPES OF ANALYSIS

— Descriptive: Summaries of traffic.

— Diagnostic: Cause/effect relationships.

— Predictive: Forecast future behavior.

— Prescriptive: Recommend actions to prevent issues.

WIRESHARK ESSENTIALS

Interface

— Packet List (Orange): Overview

— Packet Details (Blue): Protocol breakdown

— Packet Bytes (Green): Raw hex/ASCII data

Packet Structure

— Frame (Layer 2)

— Ethernet (Layer 2)

— IP (Layer 3)

— TCP/UDP (Layer 4)

— Data (Layer 7)

Filters

— Capture Filters: Apply before capturing → reduces data

Example: tcp port 80

— Display Filters: Apply after capturing → refine view

Example: http.request.method == GET

USEFUL FILTER EXAMPLES

Capture Filters

host x.x.x.x

net x.x.x.x/24

port 80

not port 443

Display Filters

ip.src == x.x.x.x

tcp.port == 443

dns / ftp / arp

tcp.port != 22

WIRESHARK STATISTICS

— Shows top talkers, protocol breakdowns, conversations, and detailed traffic reports.

FOLLOWING TCP STREAMS

— Reconstructs entire TCP conversations (text, images, files).

— Right-click packet → Follow → TCP Stream.

CONCLUSION

— Network traffic analysis is essential for security, troubleshooting, and monitoring.

— Tools like Wireshark provide deep inspection and targeted filtering for effective analysis.
