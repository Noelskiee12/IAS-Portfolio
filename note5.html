AIS NOTES

CWE-79: CROSS-SITE SCRIPTING (XSS)

— Client-side code injection where attackers run malicious scripts in users’ browsers, stealing cookies, session tokens, or performing actions on behalf of victims.

TYPES OF XSS ATTACKS

Reflected XSS (Non-Persistent / Type 1)

— Payload is sent in a request (URL/form) and immediately reflected in the response.

Example: search page echoes ?q=<script>…</script>

Stored XSS (Persistent / Type 2)

— Malicious script is saved on the server (comments, profiles) and served to all users later, enabling widespread exploitation whenever content is viewed.

DOM-Based XSS (Type 0)

— Vulnerability arises from insecure client-side JavaScript that writes attacker-controlled data into the DOM without server-side involvement.

Example: location.hash used unsafely

Universal XSS (uXSS)

— Browser or plugin-level flaw allowing persistent script execution across sites, bypassing site-specific protections and affecting many pages/users.

Self-XSS

— Social-engineering attack where users are tricked into pasting malicious code into their browser console, executing attacker scripts in their own session.

ATTACK CONSEQUENCES

— Session Hijacking / Account Takeover: Attackers steal cookies or session tokens, allowing them to log in as victims and fully control accounts.

— Data Theft / Information Exposure: Sensitive user data (emails, messages, tokens) can be read and sent to an attacker’s server.

— Phishing and Credential Theft: Fake login forms or pop-ups trick users into entering their usernames and passwords.

— Unauthorized Actions (CSRF-like Attacks): Malicious scripts perform actions like sending messages, transferring funds, or changing passwords as the victim.

— Malware Distribution / Redirects: Attackers inject code that redirects users to malicious websites or downloads harmful files.

— Financial and Reputational Damage: Stolen data, fraud, and public exposure cause loss of trust, financial loss, and possible legal penalties.

PREVENTION STRATEGIES

— Output-encode/escape contexts (HTML, JS, URL), use Content Security Policy, validate/whitelist inputs, sanitize user content, use frameworks that auto-escape, and minimize client-side DOM sinks.

Session Hijacking

— Use HttpOnly cookies and Content Security Policy (CSP).

Data Theft

— Validate and escape all user input.

Phishing

— Don’t display raw user input; auto-escape output.

Unauthorized Actions

— Use CSRF tokens and input validation.

Malware Redirects

— Block inline scripts and limit script sources (CSP).

Financial Damage

— Follow secure coding practices, test often, and patch quickly.
