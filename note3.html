AIS NOTES

CWE-78: OS COMMAND INJECTION

— An attacker injects malicious shell commands into an application that runs system commands, enabling remote code execution, data theft, or system compromise.

Example: ; rm -rf /, && nc -e /bin/sh

ATTACK MECHANISM

— Attackers use shell metacharacters (;, &&, |, `, $(), >, <) to terminate or chain commands or perform redirection, turning benign input into executable OS commands.

Example: user; malicious

MAIN ATTACK VARIANTS

Direct Injection

— User input executed immediately.

Blind Injection

— No output, attacker infers success via timing.

Chained/Piped

— Combine commands using | or &&.

Second-Order Injection

— Stored input executes later when triggered.

MITIGATION STRATEGIES

— Use allow lists and avoid direct shell invocation.

— Use safe APIs (parameterized exec libraries).

— Escape inputs and validate/normalize input.

— Enforce least privilege and run services in isolated containers/VMs.

— Example: call command APIs with argument arrays instead of a shell string.
