AIS NOTES

CWE-35: PATH TRAVERSAL

— A security vulnerability where attackers access files or directories outside the intended folder. This happens when applications build file paths using unsanitized user input. Attackers use sequences like ../ or encoded versions to escape restricted directories.

HOW FILE SYSTEMS WORK

— File systems form a tree structure.

. = current directory

.. = parent directory

— Attackers exploit .. to move upward in the file system.

DIRECTORY TRAVERSAL ATTACK BASICS

— Attacker provides malicious file path input.

— Application uses this input to build a file path.

— Missing or weak validation allows navigation outside the safe directory.

Example: /safedir/../etc/passwd → /etc/passwd

ATTACK TECHNIQUES

— Simple: ../

— Alternate separators: /, \

— Encoded: %2e%2e%2f, %252e%252e%252f

— Unicode: %u002e%u002e%u002f

— Mangled paths: ....//...

— NULL byte injection: %00

CONSEQUENCES

— Unauthorized access to sensitive files (config, passwords, source code).

— File modification or deletion → system disruption.

— Further attacks: information gathered may lead to full system compromise.

PREVENTION METHODS

— Validate & sanitize input (allowlist approach).

— Allowlist files/directories explicitly.

— Do not use user input directly in file system paths.

— Use path normalization to remove traversal sequences.

— Apply least privilege access to file systems.

— Use isolated environments like containers or chroot.
