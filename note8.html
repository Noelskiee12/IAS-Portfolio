EMAIL SECURITY FUNDAMENTALS

What is Email?

— Email is the digital exchange of messages over a network. It works through servers and protocols like SMTP, DNS, MX, and user inbox systems.

How Email Works (Simple Flow)

— Compose (email client)

— Send via SMTP server

— DNS lookup finds destination mail server

— MX server receives

— Delivered to recipient’s inbox

Core Email Security Principles

— Confidentiality: Only authorized users can read emails.

— Integrity: Prevents tampering.

— Availability: Email must be accessible.

— Authentication: Verifies the sender.

— Non-repudiation: Sender cannot deny sending the email.

Email Encryption

— Protects sensitive information and prevents unauthorized access.

— Secures data in transit and at rest.

— Helps with compliance and trust.

S/MIME (Secure/Multipurpose Internet Mail Extensions)

— Uses digital certificates issued by trusted CAs.

— Provides encryption (confidentiality) and digital signatures (authenticity + integrity).

— Built into major email clients (Outlook, Apple Mail, Thunderbird).

OpenPGP

— Open-source email encryption standard.

— Uses public/private key pairs, with users managing their own keys.

— Works across all platforms, ideal for individuals or privacy-focused users.

S/MIME vs OpenPGP (Quick Comparison)

— Trust Model: S/MIME → CA-based hierarchy; OpenPGP → Web of Trust / keyservers

— Key Management: S/MIME → Centralized; OpenPGP → User-controlled

— Ease of Use: S/MIME → More complex; OpenPGP → Generally easier

— Best For: S/MIME → Enterprises; OpenPGP → Individuals / webmail users

— Both support encryption and digital signatures.

Common Email Threats

— Phishing: Generic deception

— Spear phishing: Targeted attacks

— Malware: Attachments or links

— Email spoofing: Fake sender identity

— BEC (Business Email Compromise): Fraud via compromised accounts

— Social engineering: Psychological manipulation

— Zero-day attacks: Exploit unknown vulnerabilities

Email Header Analysis

— Used to verify sender authenticity and detect spoofing.

Key Authentication Mechanisms:

— SPF: Validates sending IP address

— DKIM: Cryptographic signature verification

— DMARC: Policy + reporting for SPF/DKIM failures

— Header checks help reveal fake or malicious emails.

Message Header Analyzer Tool

— Converts complex headers into readable format.

— Shows SPF/DKIM/DMARC pass/fail.

— Displays all server hops.

— Great for quick phishing analysis.

REMnux Platform (For Malware & Email Analysis)

— A Linux toolkit for examining malicious email artifacts.

— Tools include:

emldump – Analyze .eml files

msg-extractor – Extract .msg email files

msgconvert – Convert MSG → MBOX

mail-parser – Parse raw email content
